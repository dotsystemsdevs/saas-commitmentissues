# Contributing

Thanks for helping make dead repo certificates even better.

## Quick start

```bash
npm install
npm run dev
```

## Branch and PR

- Create a branch: `feat/short-description` or `fix/short-description`
- Keep PRs focused on one change
- Include before/after screenshots for UI changes

## Good first issues

Look for issues labeled `good first issue` and `help wanted`.

## Ground rules

- Keep the product fast and shareable
- Avoid feature bloat (no auth, no database migrations, no SaaS complexity)
- Add or update tests when touching scoring logic
